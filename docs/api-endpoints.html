Conventions & Headers

ItemNotes
Auth headerAuthorization: Bearer <sanctum-token>
Accept headerAccept: application/json (required so validation returns 422, not a web redirect).
File uploadsUse multipart/form-data. For PUT routes, POST with _method=PUT. Do not set Content-Type manually.
File URLsReturned via profile_image_url, documents[].file_url, company_logo_url. Served through public/storage symlink (php artisan storage:link).
PermissionsSpatie spatie/laravel-permission. Seed via php artisan db:seed --class=RolesAndPermissionsSeeder.
Error shapes{ "success": false, "message": "..." }, with errors object on 422 validation failures.
