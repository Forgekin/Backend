Part II · Payloads

1 · Conventions & common headers

Every JSON request:

Content-Type: application/json
Accept: application/json

Authenticated requests add:

Authorization: Bearer <sanctum-token>

File uploads use multipart/form-data. For endpoints that expect PUT, POST with _method=PUT — PHP does not parse multipart on real PUT.

Standard error shapes
// 401 — missing / invalid token
{ "message": "Unauthenticated." }

// 403 — authenticated but not authorized
{ "success": false, "message": "Unauthorized." }

// 404 — model not found
{ "message": "No query results for model [App\\Models\\...] 999" }

// 422 — validation
{
  "message": "The email has already been taken.",
  "errors": { "email": ["The email has already been taken."] }
}

// 429 — rate limit
{ "message": "Too Many Attempts." }
